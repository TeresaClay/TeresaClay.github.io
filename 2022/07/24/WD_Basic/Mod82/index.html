    fvSchemes

    • phi explanation (Divergence schemes)
      • incompressible solver: volumetrix flux of velocity on the cell faces: $\phi=\boldsymbol{u}_f\cdot \boldsymbol{S}_f$
      • mass flux for compressible flows: $\phi=\rho_f(\boldsymbol{u}_f\cdot \boldsymbol{S}_f)$

    Turbulence models
    RASModel kOmega2006 good model and results
    RASModel realizableKE good model and results
    system rotation, concetrated vorticies
    RASModel RNGkEpsilon good model and results
    system rotation, concentrated vorticies
    RASModel kEpsilon good wall modeling model
    Here we are applying it into a wall resolving meshes.
    Therefore we get wrong solution.
    RASModel kOmegaSST most general; Aber nicht unbedingt die beste Lösung
    RASModel ShihQuadraticKE LRN-KE Nonlinear
    very good mode,it is a k epsion, wall resolving
    RASModel SpalartAllmaras good results
    RASModel LaunderSharmaKE Issues with BCs and model
    RASModel LamBremhorstKE Issues with BCs and model
    diverging; negative value of epsilon

    BC for walls
    $k$
    kqRWallFunction + small value best for LRN (wave resolving) and HRN (wall modeling)
    kLowReWallFunction + small value pecific for wall resolving (y+<5 in whole domain)
    $\epsilon$
    epsilonWallFunction + small value This does not mean you’re using wall function. OpenFoam will
    compute y+ and choose the correct formulation based on y+.
    Therefore, LRN-HRN
    $\omega$
    omegaWallFunction + large value LRN-HRN. The only option you have.
    Large value since we use the inverse.
    nut
    nutUWallFunction + small value y+; best for HRN
    Can be applied to SA
    nutkWallFunction + small value y*; best for HRN
    Wrong (diverge) with SA which doesn’t produce $k$
    nutUSpaldingWallFunction + small value The most general one
    Can be applied to SA
    nutLowReWallFunction + small value best for LRN
    nuTilda
    fixed Valu
    valu uniform 0    		 the only option

divSchemes 
{
    default         none;
    div(phi,U)      bounded Gauss linearUpwind default; // always recommended
/*
//You might get unbounded or oscilatted quantities using this setup
// For convection, have to be at least 2.nd order accuracy. linearUpwind is ok  
    div(phi,k)      bounded Gauss linearUpwind default; 
    div(phi,omega)  bounded Gauss linearUpwind default; 
    div(phi,epsilon)  bounded Gauss linearUpwind default; 
    div(phi,nuTilda) bounded Gauss linearUpwind default;
*/
  // It is acceptable to use upwind. It doen't affect too much the solution in turbulence
  // Tubulence is a diffusive process. You are enhencing the diffusion using upwind
    div(phi,k)      bounded Gauss upwind;
    div(phi,omega)  bounded Gauss upwind;     // do not use Gauss linearUpwind
    div(phi,epsilon)  bounded Gauss upwind;
    div(phi,nuTilda) bounded Gauss upwind;


    div(nonlinearStress) Gauss linear;
    div((nuEff*dev2(T(grad(U))))) Gauss linear;
}
